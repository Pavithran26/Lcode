# 2492. Minimum Score of a Path Between Two Cities

**Difficulty:** Medium  
**Topics:** Graph, BFS, DFS, Union Find

---

## Problem Statement

You are given a positive integer `n` representing `n` cities numbered from `1` to `n`. You are also given a 2D array `roads` where `roads[i] = [ai, bi, distancei]` indicates that there is a **bidirectional** road between cities `ai` and `bi` with a distance equal to `distancei`. The cities graph is **not necessarily connected**.

The **score** of a path between two cities is defined as the **minimum distance** of a road in this path.

Return the **minimum possible score** of a path between cities `1` and `n`.

**Note:**

- A path is a sequence of roads between two cities.
- It is **allowed** for a path to contain the same road multiple times, and you can visit cities `1` and `n` multiple times along the path.
- The test cases are generated such that there is **at least one path** between `1` and `n`.

---

## Examples

### Example 1:

```
Input: n = 4, roads = [[1,2,9],[2,3,6],[2,4,5],[1,4,7]]
Output: 5
```

**Explanation:**  
The path from city 1 to 4 with the minimum score is: `1 -> 2 -> 4`.  
The score of this path is `min(9,5) = 5`.  
It can be shown that no other path has a smaller score.

### Example 2:

```
Input: n = 4, roads = [[1,2,2],[1,3,4],[3,4,7]]
Output: 2
```

**Explanation:**  
The path from city 1 to 4 with the minimum score is: `1 -> 2 -> 1 -> 3 -> 4`.  
The score of this path is `min(2,2,4,7) = 2`.

---

## Constraints

- `2 <= n <= 10^5`
- `1 <= roads.length <= 10^5`
- `roads[i].length == 3`
- `1 <= ai, bi <= n`
- `ai != bi`
- `1 <= distancei <= 10^4`
- There are no repeated edges.
- There is at least one path between `1` and `n`.

---

## Intuition & Approach

### Key Insight
The score of a path is the **minimum edge weight** along that path.  
Because we are allowed to revisit nodes and edges, we are not restricted to simple paths. This means we can traverse any edge in the **same connected component** as city `1` and then return to city `1` before finally going to `n`.

Since `1` and `n` are guaranteed to be connected, they belong to the same connected component. Therefore, the problem reduces to:

> Find the smallest edge weight among **all edges** that belong to the connected component containing city `1`.

### Why is this correct?
- Any edge in the component of `1` can be included in some walk from `1` to `n` (go from `1` to that edge, cross it, come back, then continue to `n`).
- The minimum score of any path will be the minimum edge weight we decide to include in the walk.
- To minimize the score, we include the globally smallest edge in the component, making it the bottleneck. Conversely, we cannot use edges outside the component because they are unreachable from `1`.

### Algorithm
1. Build an adjacency list for the graph.
2. Traverse the connected component of city `1` using **BFS** or **DFS**.
3. Keep track of the minimum edge weight encountered during the traversal.
4. Return the minimum weight found.

---

## Complexity Analysis

- **Time Complexity:** `O(n + m)` where `m` is the number of roads. We visit each node and edge in the component once.
- **Space Complexity:** `O(n + m)` for the adjacency list and the visited array.

---

## Solution Code (Python)

```python
from collections import deque
from typing import List

class Solution:
    def minScore(self, n: int, roads: List[List[int]]) -> int:
        # Step 1: Build graph adjacency list
        graph = [[] for _ in range(n + 1)]
        for u, v, w in roads:
            graph[u].append((v, w))
            graph[v].append((u, w))
        
        # Step 2: BFS to traverse the connected component of city 1
        visited = [False] * (n + 1)
        visited[1] = True
        queue = deque([1])
        
        min_edge = float('inf')
        
        while queue:
            node = queue.popleft()
            for neighbor, weight in graph[node]:
                # Update minimum edge weight in the component
                min_edge = min(min_edge, weight)
                
                if not visited[neighbor]:
                    visited[neighbor] = True
                    queue.append(neighbor)
        
        return min_edge
```

## Alternative Approach (DFS - Recursive)

```python
from typing import List

class Solution:
    def minScore(self, n: int, roads: List[List[int]]) -> int:
        graph = [[] for _ in range(n + 1)]
        for u, v, w in roads:
            graph[u].append((v, w))
            graph[v].append((u, w))
        
        visited = [False] * (n + 1)
        self.min_edge = float('inf')
        
        def dfs(node):
            visited[node] = True
            for neighbor, weight in graph[node]:
                self.min_edge = min(self.min_edge, weight)
                if not visited[neighbor]:
                    dfs(neighbor)
        
        dfs(1)
        return self.min_edge
```

---

## Alternative Approach (Union-Find)

Although BFS/DFS is simpler, we can also solve this using Union-Find:

1. Union all connected cities.
2. Find all edges where both endpoints are in the same set as city `1`.
3. Return the minimum weight among those edges.

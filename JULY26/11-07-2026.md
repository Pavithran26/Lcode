# LeetCode 2685: Count the Number of Complete Components

## Problem Statement

You are given an integer `n`. There is an undirected graph with `n` vertices, numbered from `0` to `n - 1`. You are given a 2D integer array `edges` where `edges[i] = [ai, bi]` denotes that there exists an undirected edge connecting vertices `ai` and `bi`.

Return the number of **complete connected components** of the graph.

### Definitions

- **Connected Component**: A subgraph of a graph in which there exists a path between any two vertices, and no vertex of the subgraph shares an edge with a vertex outside of the subgraph.

- **Complete Component**: A connected component is said to be **complete** if there exists an edge between every pair of its vertices.

## Examples

### Example 1

**Input:**
```
n = 6
edges = [[0,1],[0,2],[1,2],[3,4]]
```

**Output:**
```
3
```

**Explanation:** 
From the picture above, one can see that all of the components of this graph are complete.
- Component 1: {0, 1, 2} - Complete (triangle)
- Component 2: {3, 4} - Complete (edge between 3 and 4)
- Component 3: {5} - Complete (single vertex with no edges)

### Example 2

**Input:**
```
n = 6
edges = [[0,1],[0,2],[1,2],[3,4],[3,5]]
```

**Output:**
```
1
```

**Explanation:**
- Component {0, 1, 2} is complete since there is an edge between every pair of vertices.
- Component {3, 4, 5} is **not** complete since there is no edge between vertices 4 and 5.
- Thus, the number of complete components is 1.

## Constraints

```
1 <= n <= 50
0 <= edges.length <= n * (n - 1) / 2
edges[i].length == 2
0 <= ai, bi <= n - 1
ai != bi
There are no repeated edges.
```

## Solution Approaches

### Approach 1: DFS + Edge Count Verification

**Idea:**
1. Find all connected components using DFS
2. For each component, check if it's complete by verifying that the number of edges in the component equals the number of possible edges

For a complete graph with `k` vertices, the number of edges should be `k * (k - 1) / 2`.

**Code:**
```java
class Solution {
    public int countCompleteComponents(int n, int[][] edges) {
        // Build adjacency list
        List<List<Integer>> graph = new ArrayList<>();
        for (int i = 0; i < n; i++) {
            graph.add(new ArrayList<>());
        }
        
        for (int[] edge : edges) {
            graph.get(edge[0]).add(edge[1]);
            graph.get(edge[1]).add(edge[0]);
        }
        
        boolean[] visited = new boolean[n];
        int completeComponents = 0;
        
        // Find all connected components
        for (int i = 0; i < n; i++) {
            if (!visited[i]) {
                List<Integer> component = new ArrayList<>();
                dfs(i, graph, visited, component);
                
                // Check if this component is complete
                int vertices = component.size();
                int expectedEdges = vertices * (vertices - 1) / 2;
                int actualEdges = 0;
                
                // Count edges within this component
                for (int v : component) {
                    for (int neighbor : graph.get(v)) {
                        if (v < neighbor && component.contains(neighbor)) {
                            actualEdges++;
                        }
                    }
                }
                
                if (actualEdges == expectedEdges) {
                    completeComponents++;
                }
            }
        }
        
        return completeComponents;
    }
    
    private void dfs(int node, List<List<Integer>> graph, boolean[] visited, List<Integer> component) {
        visited[node] = true;
        component.add(node);
        
        for (int neighbor : graph.get(node)) {
            if (!visited[neighbor]) {
                dfs(neighbor, graph, visited, component);
            }
        }
    }
}
```

### Approach 2: DFS + Degree Check (More Efficient)

**Idea:**
For a complete component with `k` vertices, every vertex must have degree exactly `(k - 1)` within the component.

**Code:**
```java
class Solution {
    public int countCompleteComponents(int n, int[][] edges) {
        // Build adjacency list
        List<Set<Integer>> graph = new ArrayList<>();
        for (int i = 0; i < n; i++) {
            graph.add(new HashSet<>());
        }
        
        for (int[] edge : edges) {
            graph.get(edge[0]).add(edge[1]);
            graph.get(edge[1]).add(edge[0]);
        }
        
        boolean[] visited = new boolean[n];
        int completeComponents = 0;
        
        for (int i = 0; i < n; i++) {
            if (!visited[i]) {
                List<Integer> component = new ArrayList<>();
                dfs(i, graph, visited, component);
                
                // Check if complete using degree property
                int k = component.size();
                boolean isComplete = true;
                
                for (int vertex : component) {
                    // In a complete component, each vertex should connect to all other k-1 vertices
                    int degree = 0;
                    for (int neighbor : graph.get(vertex)) {
                        if (component.contains(neighbor)) {
                            degree++;
                        }
                    }
                    if (degree != k - 1) {
                        isComplete = false;
                        break;
                    }
                }
                
                if (isComplete) {
                    completeComponents++;
                }
            }
        }
        
        return completeComponents;
    }
    
    private void dfs(int node, List<Set<Integer>> graph, boolean[] visited, List<Integer> component) {
        visited[node] = true;
        component.add(node);
        
        for (int neighbor : graph.get(node)) {
            if (!visited[neighbor]) {
                dfs(neighbor, graph, visited, component);
            }
        }
    }
}
```

### Approach 3: Union-Find (Disjoint Set Union)

**Idea:**
1. Use DSU to find all connected components
2. For each component, count vertices and edges
3. A component is complete if `edges_in_component == vertices * (vertices - 1) / 2`

**Code:**
```java
class Solution {
    class DSU {
        int[] parent;
        int[] rank;
        
        public DSU(int n) {
            parent = new int[n];
            rank = new int[n];
            for (int i = 0; i < n; i++) {
                parent[i] = i;
            }
        }
        
        public int find(int x) {
            if (parent[x] != x) {
                parent[x] = find(parent[x]);
            }
            return parent[x];
        }
        
        public void union(int x, int y) {
            int px = find(x);
            int py = find(y);
            if (px == py) return;
            
            if (rank[px] < rank[py]) {
                parent[px] = py;
            } else if (rank[px] > rank[py]) {
                parent[py] = px;
            } else {
                parent[py] = px;
                rank[px]++;
            }
        }
    }
    
    public int countCompleteComponents(int n, int[][] edges) {
        DSU dsu = new DSU(n);
        
        // Union all edges
        for (int[] edge : edges) {
            dsu.union(edge[0], edge[1]);
        }
        
        // Count vertices in each component
        Map<Integer, Integer> vertexCount = new HashMap<>();
        for (int i = 0; i < n; i++) {
            int root = dsu.find(i);
            vertexCount.put(root, vertexCount.getOrDefault(root, 0) + 1);
        }
        
        // Count edges in each component
        Map<Integer, Integer> edgeCount = new HashMap<>();
        for (int[] edge : edges) {
            int root = dsu.find(edge[0]);
            edgeCount.put(root, edgeCount.getOrDefault(root, 0) + 1);
        }
        
        // Check completeness
        int completeComponents = 0;
        for (int root : vertexCount.keySet()) {
            int vertices = vertexCount.get(root);
            int expectedEdges = vertices * (vertices - 1) / 2;
            int actualEdges = edgeCount.getOrDefault(root, 0);
            
            if (actualEdges == expectedEdges) {
                completeComponents++;
            }
        }
        
        return completeComponents;
    }
}
```

## Complexity Analysis

| Approach | Time Complexity | Space Complexity |
|----------|----------------|------------------|
| DFS + Edge Count | O(V + E) | O(V + E) |
| DFS + Degree Check | O(V + E) | O(V + E) |
| Union-Find | O(V + E × α(V)) | O(V + E) |

Where:
- V = number of vertices (n)
- E = number of edges
- α = inverse Ackermann function (practically constant)

---

## Key Insights

1. **Single vertex components** are always complete (0 edges, 0 expected edges).

2. **Complete graph property**: For a component with k vertices to be complete, it must have exactly k(k-1)/2 edges.

3. **Alternative check**: In a complete component, every vertex must have degree (k-1) within the component.

4. **Union-Find approach** is elegant because it naturally groups vertices into components without explicit DFS traversal.

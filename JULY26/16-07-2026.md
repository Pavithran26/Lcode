# 3867. Sum of GCD of Formed Pairs

## Problem Statement

You are given an integer array `nums` of length `n`.

Construct an array `prefixGcd` where for each index `i`:

- Let `mxi = max(nums[0], nums[1], ..., nums[i])`.
- `prefixGcd[i] = gcd(nums[i], mxi)`.

After constructing `prefixGcd`:

1. Sort `prefixGcd` in non-decreasing order.
2. Form pairs by taking the **smallest** unpaired element and the **largest** unpaired element. Repeat this process until no more pairs can be formed.
3. For each formed pair, compute the GCD of the two elements.
4. If `n` is odd, the middle element in the sorted array remains unpaired and is ignored.

Return an integer denoting the **sum** of the GCD values of all formed pairs.

> **Note:** `gcd(a, b)` denotes the greatest common divisor of `a` and `b`.

---

## Examples

### Example 1

**Input:**
```
nums = [2, 6, 4]
```

**Output:**
```
2
```

**Explanation:**

Construct `prefixGcd`:

| i | nums[i] | mxi | prefixGcd[i] |
|---|---------|-----|--------------|
| 0 | 2       | 2   | 2            |
| 1 | 6       | 6   | 6            |
| 2 | 4       | 6   | 2            |

`prefixGcd = [2, 6, 2]`. After sorting: `[2, 2, 6]`.

- Pair smallest and largest: `gcd(2, 6) = 2`.
- Middle element `2` is ignored.

Sum = `2`.

---

### Example 2

**Input:**
```
nums = [3, 6, 2, 8]
```

**Output:**
```
5
```

**Explanation:**

Construct `prefixGcd`:

| i | nums[i] | mxi | prefixGcd[i] |
|---|---------|-----|--------------|
| 0 | 3       | 3   | 3            |
| 1 | 6       | 6   | 6            |
| 2 | 2       | 6   | 2            |
| 3 | 8       | 8   | 8            |

`prefixGcd = [3, 6, 2, 8]`. After sorting: `[2, 3, 6, 8]`.

- Pair 1: `gcd(2, 8) = 2`
- Pair 2: `gcd(3, 6) = 3`

Sum = `2 + 3 = 5`.

---

## Constraints

- `1 <= n == nums.length <= 10^5`
- `1 <= nums[i] <= 10^9`

---

## Solution Approach

### Intuition

1. **Build `prefixGcd`:**
   - Keep track of the maximum element seen so far (`mxi`).
   - For each index `i`, compute `gcd(nums[i], mxi)` and store it.

2. **Sort the array** to arrange elements in non-decreasing order.

3. **Pair and Sum:**
   - Use two pointers: one starting from the smallest (`left`) and one from the largest (`right`).
   - For each pair `(left, right)`, compute `gcd(prefixGcd[left], prefixGcd[right])` and add to the answer.
   - Move pointers inward until they cross.
   - If `n` is odd, the middle element is skipped automatically because we only loop for `n/2` pairs.

### Complexity

- **Time Complexity:** `O(n log n)` – due to sorting the `prefixGcd` array.  
  Building the array takes `O(n)`, and each `gcd` computation is `O(log max)`.
- **Space Complexity:** `O(n)` – for storing the `prefixGcd` array.

---

## Solution Code (Java)

```java
class Solution {
    public long gcdSum(int[] nums) {
        int n = nums.length;
        int[] prefixGcd = new int[n];
        int max = 0;

        // Step 1: Build prefixGcd array
        for (int i = 0; i < n; i++) {
            max = Math.max(max, nums[i]);
            prefixGcd[i] = gcd(nums[i], max);
        }

        // Step 2: Sort the array
        java.util.Arrays.sort(prefixGcd);

        // Step 3: Pair smallest with largest and sum their GCD
        long ans = 0;
        for (int i = 0; i < n / 2; i++) {
            ans += gcd(prefixGcd[i], prefixGcd[n - 1 - i]);
        }

        return ans;
    }

    // Helper function to compute GCD using Euclidean algorithm
    private int gcd(int a, int b) {
        while (b != 0) {
            int temp = a % b;
            a = b;
            b = temp;
        }
        return a;
    }
}
```

| Step | Operation                                 | Result                  |

| 2    | Sort                                      | `[2, 3, 6, 8]`          |
| 3    | Pair `(2, 8)` → `gcd(2,8)=2`              | ans = 2                 |
| 4    | Pair `(3, 6)` → `gcd(3,6)=3`              | ans = 5                 |
| 5    | Return                                    | 5                       |

## Key Points to Remember

- The middle element is automatically ignored because we only iterate for `n/2` pairs.
- The GCD helper uses the Euclidean algorithm for efficiency.
- The answer can be up to `n/2 * 10^9`, so use `long` to avoid overflow.




# LeetCode 3620. Network Recovery Pathways (Hard)

## Problem Description

You are given a directed acyclic graph of `n` nodes numbered from `0` to `n-1`. This is represented by a 2D array `edges` of length `m`, where `edges[i] = [ui, vi, costi]` indicates a one‑way communication from node `ui` to node `vi` with a recovery cost of `costi`.

Some nodes may be offline. You are given a boolean array `online` where `online[i] = true` means node `i` is online. Nodes `0` and `n-1` are always online.

A path from `0` to `n-1` is valid if:

- All intermediate nodes on the path are online.
- The total recovery cost of all edges on the path does not exceed `k`.

For each valid path, define its **score** as the minimum edge‑cost along that path.

Return the **maximum path score** (i.e., the largest minimum-edge cost) among all valid paths. If no valid path exists, return `-1`.

---

## Examples

### Example 1

```
Input: edges = [[0,1,5],[1,3,10],[0,2,3],[2,3,4]], 
       online = [true,true,true,true], 
       k = 10
Output: 3
```

**Explanation:**
- Path `0 → 1 → 3`: total cost = 5 + 10 = 15 > 10 → invalid.
- Path `0 → 2 → 3`: total cost = 3 + 4 = 7 ≤ 10 → valid.  
  Minimum edge‑cost = min(3,4) = 3.
- No other valid paths → answer = `3`.

### Example 2

```
Input: edges = [[0,1,7],[1,4,5],[0,2,6],[2,3,6],[3,4,2],[2,4,6]], 
       online = [true,true,true,false,true], 
       k = 12
Output: 6
```

**Explanation:**
- Node `3` is offline → any path through it is invalid.
- Path `0 → 1 → 4`: total cost = 7 + 5 = 12 ≤ 12, score = min(7,5) = 5.
- Path `0 → 2 → 4`: total cost = 6 + 6 = 12 ≤ 12, score = min(6,6) = 6.
- Maximum score among valid paths = `6`.

---

## Constraints

- `n == online.length`
- `2 <= n <= 5 * 10^4`
- `0 <= m == edges.length <= min(10^5, n * (n - 1) / 2)`
- `edges[i] = [ui, vi, costi]`
- `0 <= ui, vi < n`
- `ui != vi`
- `0 <= costi <= 10^9`
- `0 <= k <= 5 * 10^13`
- `online[i]` is either `true` or `false`, and both `online[0]` and `online[n-1]` are `true`.
- The given graph is a directed acyclic graph (DAG).

---

## Solution Approach

### Key Idea

We need to maximize the **minimum edge cost** on a path from `0` to `n-1` subject to:
1. All intermediate nodes are online.
2. Total path cost ≤ `k`.

This is a **maximin** problem with a budget constraint. We can binary search on the answer (the minimum edge cost `x`) and check if there exists a valid path where every edge has cost **≥** `x` and the sum of those costs ≤ `k`.

### Feasibility Check (`feasible(x)`)

Given a threshold `x`:
- Only consider edges with `cost >= x`.
- Only traverse through online nodes.
- Compute the shortest path (minimum total cost) from `0` to `n-1` using only such edges.
- If the minimum total cost ≤ `k`, then there exists a valid path with score at least `x`.

Since the graph is a **DAG**, we can compute the shortest path in **O(n + m)** using topological order DP (instead of Dijkstra).

### Binary Search

- **Lower bound**: `0` (all edges are allowed, since costs are non-negative).
- **Upper bound**: `max cost` among all edges.
- We binary search the largest `x` for which `feasible(x)` is true.

### Edge Case – No Valid Path
If even `x = 0` fails (i.e., no path using only online nodes with total cost ≤ `k`), then the answer is `-1`.

---

## Complexity

- **Topological sort**: `O(n + m)`
- **Feasibility check**: `O(n + m)` (DP over topological order)
- **Binary search range**: `O(log(maxCost))` ≤ about 30 iterations.
- **Overall**: **O((n + m) log C)**, where `C = 10^9`.

## Java Code

```java
import java.util.*;

class Solution {
    public int findMaxPathScore(int[][] edges, boolean[] online, long k) {
        int n = online.length;
        List<int[]>[] graph = new List[n];
        for (int i = 0; i < n; i++) {
            graph[i] = new ArrayList<>();
        }

        // Build graph and compute indegree for topological sort
        int[] indegree = new int[n];
        long maxCost = 0;

        for (int[] e : edges) {
            int u = e[0], v = e[1], c = e[2];
            graph[u].add(new int[]{v, c});
            indegree[v]++;
            maxCost = Math.max(maxCost, c);
        }

        // Topological order (Kahn's algorithm)
        Queue<Integer> q = new ArrayDeque<>();
        for (int i = 0; i < n; i++) {
            if (indegree[i] == 0) q.offer(i);
        }

        List<Integer> topo = new ArrayList<>(n);
        while (!q.isEmpty()) {
            int u = q.poll();
            topo.add(u);
            for (int[] edge : graph[u]) {
                int v = edge[0];
                indegree[v]--;
                if (indegree[v] == 0) q.offer(v);
            }
        }

        // If even with threshold 0 no path exists, return -1
        if (!feasible(0L, n, graph, online, topo, k)) {
            return -1;
        }

        // Binary search the maximum possible minimum edge cost
        long lo = 0, hi = maxCost;
        while (lo < hi) {
            long mid = (lo + hi + 1) / 2;
            if (feasible(mid, n, graph, online, topo, k)) {
                lo = mid;
            } else {
                hi = mid - 1;
            }
        }

        return (int) lo;
    }

    private boolean feasible(
        long x,
        int n,
        List<int[]>[] graph,
        boolean[] online,
        List<Integer> topo,
        long k
    ) {
        final long INF = Long.MAX_VALUE / 4;
        long[] dist = new long[n];
        Arrays.fill(dist, INF);
        dist[0] = 0;

        for (int u : topo) {
            // Skip if unreachable or offline
            if (dist[u] >= INF || !online[u]) continue;

            for (int[] edge : graph[u]) {
                int v = edge[0];
                int cost = edge[1];

                // Must use edge cost >= x and destination must be online
                if (!online[v] || cost < x) continue;

                long nd = dist[u] + cost;
                if (nd < dist[v]) {
                    dist[v] = nd;
                }
            }
        }

        return dist[n - 1] <= k;
    }
}
```

---

## Explanation of the Code

1. **Graph Building**  
   We store the graph as an adjacency list of `[neighbor, cost]`. We also compute indegrees for topological sort.

2. **Topological Sort**  
   Since the graph is a DAG, we process nodes in topological order. This allows us to run DP in one pass.

3. **Feasibility Function**  
   - `dist[u]` = minimum total cost to reach `u` using edges with cost ≥ `x` and online nodes only.  
   - We iterate nodes in topological order and relax edges that meet the threshold `x` and go to an online node.  
   - At the end, we check if `dist[n-1] ≤ k`.

4. **Binary Search**  
   We search between `0` and `maxCost` for the highest `x` that is feasible.  
   - `lo` is always feasible.  
   - `hi` is the upper bound.  
   - `mid = (lo+hi+1)/2` (upper‑mid) to avoid infinite loops.

5. **Initial Check**  
   We call `feasible(0)` first. If it returns `false`, no path exists at all (even with all edges allowed), so we return `-1` immediately.

## Dry Run on Example 1

- `maxCost = 10`, binary search range `[0,10]`.
- `feasible(5)` → edges ≥5: (0→1,5), (1→3,10).  
  Dist: 0→1=5, 1→3=15 → total 15 > 10 → false.
- `feasible(3)` → edges ≥3: all edges.  
  Dist: 0→2=3, 2→3=7 → total 7 ≤ 10 → true.
- Higher thresholds fail, so answer = 3.

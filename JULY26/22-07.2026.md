# 3501. Maximize Active Section with Trade II

**Difficulty:** Hard  
**LeetCode Problem Number:** 3501  

---

## Problem Statement

You are given a binary string `s` of length `n`, where:

- `'1'` represents an **active** section.
- `'0'` represents an **inactive** section.

You can perform **at most one trade** to maximize the number of active sections in `s`. In a trade, you:

1. Convert a contiguous block of `'1'`s that is surrounded by `'0'`s to all `'0'`s.
2. Afterward, convert a contiguous block of `'0'`s that is surrounded by `'1'`s to all `'1'`s.

Additionally, you are given a 2D array `queries`, where `queries[i] = [li, ri]` represents a substring `s[li...ri]`.

For each query, determine the **maximum possible number of active sections** in `s` after making the optimal trade **on the substring** `s[li...ri]`.

Return an array `answer`, where `answer[i]` is the result for `queries[i]`.

### Notes

- For each query, treat `s[li...ri]` as if it is augmented with a `'1'` at both ends, forming  
  `t = '1' + s[li...ri] + '1'`. The augmented `'1'`s **do not** contribute to the final count.
- The queries are **independent** of each other.

---

## Constraints

- `1 <= n == s.length <= 10^5`
- `1 <= queries.length <= 10^5`
- `s[i]` is either `'0'` or `'1'`.
- `0 <= li <= ri < n`

## Examples

### Example 1

```
Input: s = "01", queries = [[0,1]]
Output: [1]
```

**Explanation:**  
There is no block of `'1'`s surrounded by `'0'`s, so no valid trade is possible. The maximum number of active sections is 1.

- **Query [1,3]**: Substring `"100"` ‚Üí Augmented `"11001"`  
  No valid trade possible ‚Üí active sections = 1.

- **Query [2,3]**: Substring `"00"` ‚Üí Augmented `"1001"`  
  No valid trade possible ‚Üí active sections = 1.

## Solution Approach

### Intuition

The trade consists of:

1. **Choose a run of '1's** (surrounded by '0's) to delete (convert to zeros).
2. **Choose a run of '0's** (surrounded by '1's) to fill (convert to ones).

The gain is the **increase** in active sections after the trade. The total after trade = `totalOnes + gain`.  
We need to compute the maximum possible gain for each query substring.

### Key Observations

- The operation can only affect **zero‚Äëruns** and the **one‚Äëruns** between them.
- For a query `[l, r]`, we consider only the zero‚Äëruns that intersect the query range.
- The best trade either:
  - **Adjacent zero‚Äëruns**: combine two neighboring zero‚Äëruns (the one‚Äërun between them is deleted, then the combined zero block is filled).  
    Gain = `len(zero_i) + len(zero_{i+1})`.
  - **Non‚Äëadjacent zero‚Äëruns**: delete a one‚Äërun and fill a distant zero‚Äërun.  
    Gain = `max_other_zero_len - len(one_run)`.

### Data Structures

- **Zero‚Äëruns** are collected with their start, end, and length.
- **One‚Äëruns** between consecutive zero‚Äëruns are stored.
- We use:
  - **Segment Tree (max)** to get the maximum zero‚Äërun length in a range.
  - **Segment Tree (min)** to get the minimum one‚Äërun length in a range.
  - **Segment Tree (max)** to get the maximum `zero_i + zero_{i+1}` in a range.
- For each query, we:
  1. Find the range of zero‚Äërun indices that intersect `[l, r]`.
  2. Evaluate candidates:
     - Boundary candidates (where the zero‚Äëruns are clipped by the query edges).
     - Internal candidates (fully inside).
  3. Use segment trees to query maximal values quickly.

### Complexity

- **Time:** `O((n + q) log n)` per query, dominated by segment tree queries.
- **Space:** `O(n)` for storing arrays and segment trees.

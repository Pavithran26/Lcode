# 3658. GCD of Odd and Even Sums

## Problem Statement

You are given an integer `n`. Your task is to compute the GCD (greatest common divisor) of two values:

- `sumOdd`: the sum of the smallest `n` positive odd numbers.
- `sumEven`: the sum of the smallest `n` positive even numbers.

Return the GCD of `sumOdd` and `sumEven`.

---

## Examples

### Example 1

**Input:** `n = 4`  
**Output:** `4`

**Explanation:**
- Sum of the first 4 odd numbers: `sumOdd = 1 + 3 + 5 + 7 = 16`
- Sum of the first 4 even numbers: `sumEven = 2 + 4 + 6 + 8 = 20`
- GCD(16, 20) = 4

---

### Example 2

**Input:** `n = 5`  
**Output:** `5`

**Explanation:**
- Sum of the first 5 odd numbers: `sumOdd = 1 + 3 + 5 + 7 + 9 = 25`
- Sum of the first 5 even numbers: `sumEven = 2 + 4 + 6 + 8 + 10 = 30`
- GCD(25, 30) = 5

---

## Constraints

- `1 <= n <= 10^100` (The mathematical solution works for any `n`.)

---

## Approach

### Mathematical Insight

1. **Sum of first `n` odd numbers:**
   - The odd numbers are `1, 3, 5, ...`
   - Their sum is a well‑known formula:  
     `sumOdd = n²`

2. **Sum of first `n` even numbers:**
   - The even numbers are `2, 4, 6, ...`
   - Their sum is:  
     `sumEven = n * (n + 1)`

3. **GCD Calculation:**
   - We need `GCD(n², n * (n + 1))`.
   - Factor out `n`:  
     `GCD(n * n, n * (n + 1)) = n * GCD(n, n + 1)`
   - Since `n` and `n + 1` are consecutive integers, they are always coprime:  
     `GCD(n, n + 1) = 1`
   - Therefore, the result is simply `n`.

### Complexity

- **Time Complexity:** `O(1)` – constant time.
- **Space Complexity:** `O(1)` – no extra space used.

---

## Solution (Java)

```java
class Solution {
    public int gcdOfOddEvenSums(int n) {
        // Sum of first n odd numbers = n^2
        // Sum of first n even numbers = n * (n + 1)
        // GCD(n^2, n(n+1)) = n, because GCD(n, n+1) = 1
        return n;
    }
}
```

# 3513. Number of Unique XOR Triplets I

## Problem Statement
You are given an integer array `nums` of length `n`, where `nums` is a **permutation** of the numbers in the range `[1, n]`.

A **XOR triplet** is defined as the XOR of three elements:
```
nums[i] XOR nums[j] XOR nums[k]
```
where `i <= j <= k`.

Return the number of **unique** XOR triplet values from all possible triplets `(i, j, k)`.

---

## Key Observations

1. Since `nums` is a permutation of `[1, n]`, every integer from `1` to `n` appears exactly once.
2. Indices can repeat (`i <= j <= k`), meaning we can pick the **same** element multiple times.
3. Thus, any triplet is equivalent to choosing three numbers `(a, b, c)` from the set `[1, n]` **with repetition allowed**.

---

## Mathematical Insight

Let:
- `p = highestPowerOfTwo <= n`  (i.e., `Integer.highestOneBit(n)`).
- Then `2p` is the **smallest power of two strictly greater than `n`**.

The set of all possible XORs of three numbers from `[1, n]` is exactly:
- All integers from `0` to `2p - 1` **when `n > 2`**.
- This gives exactly `2p` distinct values.

### Why does this hold?
- Since we have all numbers `1` through `n`, and `n >= p`, we have the number `p` (which is a power of two) and enough other numbers to generate any bit combination up to bit `p`.
- With three numbers, we can produce `0` (e.g., `a ^ a ^ a = a` for any `a`) and all values up to `2p - 1`.
- The maximum value we can get is `(2p - 1)` because no number has a bit higher than `p`.
- The constructive proof (standard for this problem) shows that every value in `[0, 2p - 1]` is achievable.

### Edge cases
- `n = 1`: only triplet is `(1,1,1) → 1`. Result = `1`.
- `n = 2`: possible values are `1` and `2`. Result = `2`.

For `n > 2`, the formula `2 * p` works perfectly.

---

## Algorithm

1. If `n <= 2`, return `n`.
2. Otherwise, compute `p = Integer.highestOneBit(n)`.
3. Return `p << 1` (which equals `2 * p`).

**Time Complexity:** `O(1)`  
**Space Complexity:** `O(1)`

---

## Java Code

```java
class Solution {
    public int uniqueXorTriplets(int[] nums) {
        int n = nums.length;
        
        // Base cases for small n
        if (n <= 2) {
            return n;
        }
        
        // Largest power of two <= n
        int highestPower = Integer.highestOneBit(n);
        
        // Smallest power of two strictly greater than n
        return highestPower << 1;
    }
}
```

## Examples

### Example 1
```
Input: nums = [1,2]
n = 2 → output = 2
```

### Example 2
```
Input: nums = [3,1,2]
n = 3 → highestPower = 2 → output = 4
```

### Example 3 (Mental)
```
Input: nums = [1,2,3,4]
n = 4 → highestPower = 4 → output = 8
```
All values `0..7` are achievable, hence 8 unique triplets.

---

## Summary

| `n` range          | Answer      |
|---------------------|-------------|
| `n = 1`             | `1`         |
| `n = 2`             | `2`         |
| `n > 2`             | `2 * highestPowerOfTwo(n)` |

This is a constant–time solution that leverages the structure of permutations and bitwise properties.

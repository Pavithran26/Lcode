# 3514. Number of Unique XOR Triplets II

## Problem Statement

Given an integer array `nums`, a **XOR triplet** is defined as the XOR of three elements:

`nums[i] XOR nums[j] XOR nums[k]` where `i <= j <= k`.

Return the number of **unique** XOR triplet values from all possible triplets `(i, j, k)`.

---

## Approach

### Key Observations

1. **Constraints**  
   - `nums.length <= 1500`
   - `nums[i] <= 1500`

   The values are small, so we can use bitsets or boolean arrays to track reachable XOR values.

2. **XOR Properties**  
   - XOR is associative and commutative.
   - The maximum possible XOR value is less than the next power of two greater than the maximum number in `nums`. Let this limit be `L`.

3. **Reducing the Problem**  
   - Since `i <= j <= k`, order does not matter because XOR is commutative and we can pick any element multiple times (due to `i <= j <= k`).
   - So, a triplet is effectively: pick any three elements (with repetition) from the set of distinct values in `nums`, and XOR them.

4. **Two-Step Computation**  
   - **Step 1:** Compute all possible XORs of **pairs** (`nums[i] ^ nums[j]`) from the distinct values.  
   - **Step 2:** XOR each pair result with every distinct value to get all possible triplet XORs.

5. **Optimization**  
   - Instead of O(n³) brute force (which would be ~1500³ = 3.3e9), we:
     - Deduplicate `nums` into a boolean array of size `L`.
     - Build a list of distinct values.
     - Compute pair XORs in O(d²), where `d` is the number of distinct values (≤ 1500).
     - Then compute triplet XORs in O(L * d) or O(number_of_pairs * d), but since we only have boolean arrays up to `L`, we can loop over the boolean pair array and the distinct values.

---

## Complexity

- **Time Complexity:**  
  - Building distinct list: O(n)  
  - Pair XOR computation: O(d²)  
  - Triplet XOR computation: O(L * d) where `L` is the power-of-two bound.  
  Since `L ≤ 2048` (because max value ≤ 1500, next power of two is 2048), and `d ≤ 1500`, this is easily manageable.

- **Space Complexity:**  
  O(L) for the boolean arrays.

---

## Java Solution

```java
class Solution {
    public int uniqueXorTriplets(int[] nums) {
        int maxVal = 0;
        for (int x : nums) maxVal = Math.max(maxVal, x);

        // Power of two strictly greater than every input value
        int limit = 1;
        while (limit <= maxVal) limit <<= 1;

        // Store distinct values
        boolean[] has = new boolean[limit];
        for (int x : nums) has[x] = true;

        int[] vals = new int[limit];
        int d = 0;
        for (int x = 0; x < limit; x++) {
            if (has[x]) vals[d++] = x;
        }

        // pairXor[p] = true if p = vals[i] ^ vals[j] for some i, j
        boolean[] pairXor = new boolean[limit];
        for (int i = 0; i < d; i++) {
            for (int j = 0; j < d; j++) {
                pairXor[vals[i] ^ vals[j]] = true;
            }
        }

        // Every triplet XOR is (vals[i] ^ vals[j]) ^ vals[k]
        boolean[] unique = new boolean[limit];
        for (int p = 0; p < limit; p++) {
            if (!pairXor[p]) continue;
            for (int i = 0; i < d; i++) {
                unique[p ^ vals[i]] = true;
            }
        }

        int ans = 0;
        for (boolean b : unique) {
            if (b) ans++;
        }
        return ans;
    }
}
```

---

## Dry Run with Example

### Example 1: `nums = [1,3]`
- `maxVal = 3` → `limit = 4`
- Distinct `vals = [1, 3]`
- Pair XORs: `1^1=0`, `1^3=2`, `3^1=2`, `3^3=0` → `{0, 2}`
- Triplet XORs:
  - `0 ^ 1 = 1`
  - `0 ^ 3 = 3`
  - `2 ^ 1 = 3`
  - `2 ^ 3 = 1` → `{1, 3}`
- Count = 2 → matches expected output.

### Example 2: `nums = [6,7,8,9]`
- `limit = 16`
- All pair XORs will generate all values from 0 to 15.
- Then triplet XORs also generate exactly the original set `{6,7,8,9}` (as per problem statement), count = 4.

---

## Why This Works

- Since `i <= j <= k`, repetition is allowed.  
- By computing all pair XORs and then XORing with every distinct element, we effectively cover all combinations of three elements (with repetition).  
- The boolean array limits the search space to the relevant bit-length, making the solution efficient for the given constraints.

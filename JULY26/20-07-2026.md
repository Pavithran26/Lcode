# 1260. Shift 2D Grid

## Problem Statement

Given a 2D grid of size `m x n` and an integer `k`. You need to shift the grid `k` times.

In one shift operation:

- Element at `grid[i][j]` moves to `grid[i][j + 1]`.
- Element at `grid[i][n - 1]` moves to `grid[i + 1][0]`.
- Element at `grid[m - 1][n - 1]` moves to `grid[0][0]`.

Return the 2D grid after applying shift operation `k` times.

### Constraints

- `m == grid.length`
- `n == grid[i].length`
- `1 <= m <= 50`
- `1 <= n <= 50`
- `-1000 <= grid[i][j] <= 1000`
- `0 <= k <= 100`

---

## Examples

### Example 1

**Input:**  
`grid = [[1,2,3],[4,5,6],[7,8,9]]`, `k = 1`  
**Output:**  
`[[9,1,2],[3,4,5],[6,7,8]]`

### Example 2

**Input:**  
`grid = [[3,8,1,9],[19,7,2,5],[4,6,11,10],[12,0,21,13]]`, `k = 4`  
**Output:**  
`[[12,0,21,13],[3,8,1,9],[19,7,2,5],[4,6,11,10]]`

### Example 3

**Input:**  
`grid = [[1,2,3],[4,5,6],[7,8,9]]`, `k = 9`  
**Output:**  
`[[1,2,3],[4,5,6],[7,8,9]]`

---

## Approach

1. **Flatten the 2D grid** into a 1D array of size `m * n`.
2. Compute the effective shift as `k % (m * n)` because shifting by the total number of elements results in the same grid.
3. For each element at position `i` (row-major order), its new position in the flattened array will be `(i + k) % total`.
4. Convert the new 1D index back to 2D coordinates using `newRow = newIndex / n` and `newCol = newIndex % n`.
5. Place the element into the result grid.

This approach avoids simulating `k` shifts and works in **O(m * n)** time and **O(m * n)** space.

---

## Complexity Analysis

- **Time Complexity:** `O(m * n)`, we iterate over every element once.
- **Space Complexity:** `O(m * n)` for the output grid (excluding the input).

---

## Java Solution

```java
class Solution {
    public List<List<Integer>> shiftGrid(int[][] grid, int k) {
        int m = grid.length;
        int n = grid[0].length;
        int total = m * n;
        
        // Only the remainder matters
        k %= total;
        
        int[][] result = new int[m][n];
        
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                // Current position in flattened 1D array
                int oldIndex = i * n + j;
                
                // New position after shifting right by k
                int newIndex = (oldIndex + k) % total;
                
                int newRow = newIndex / n;
                int newCol = newIndex % n;
                
                result[newRow][newCol] = grid[i][j];
            }
        }
        
        // Convert 2D array to List<List<Integer>>
        List<List<Integer>> output = new ArrayList<>();
        for (int[] row : result) {
            List<Integer> list = new ArrayList<>();
            for (int num : row) {
                list.add(num);
            }
            output.add(list);
        }
        
        return output;
    }
}
```

---

## Notes

- The shift operation is cyclic; elements wrap around from the last column to the next row, and from the last row to the first row.
- Using modulo arithmetic avoids the need for a queue or multiple rotations.

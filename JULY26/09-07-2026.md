
# 3532. Path Existence Queries in a Graph I

## Problem Statement

You are given an integer `n` representing the number of nodes in a graph, labeled from `0` to `n - 1`.

You are also given an integer array `nums` of length `n` sorted in **non-decreasing order**, and an integer `maxDiff`.

An **undirected edge** exists between nodes `i` and `j` if the absolute difference between `nums[i]` and `nums[j]` is at most `maxDiff` (i.e., `|nums[i] - nums[j]| <= maxDiff`).

You are also given a 2D integer array `queries`. For each `queries[i] = [ui, vi]`, determine whether there exists a path between nodes `ui` and `vi`.

Return a boolean array `answer`, where `answer[i]` is `true` if there exists a path between `ui` and `vi` in the `i`th query, and `false` otherwise.

---

## Examples

### Example 1

**Input:**
```
n = 2, nums = [1,3], maxDiff = 1, queries = [[0,0],[0,1]]
```

**Output:**
```
[true, false]
```

**Explanation:**
- Query `[0,0]`: Node 0 has a trivial path to itself → `true`.
- Query `[0,1]`: No edge exists because `|1 - 3| = 2 > maxDiff` → `false`.

---

### Example 2

**Input:**
```
n = 4, nums = [2,5,6,8], maxDiff = 2, queries = [[0,1],[0,2],[1,3],[2,3]]
```

**Output:**
```
[false, false, true, true]
```

**Explanation:**
- Query `[0,1]`: `|2 - 5| = 3 > 2` → no edge → `false`.
- Query `[0,2]`: `|2 - 6| = 4 > 2` → no edge → `false`.
- Query `[1,3]`: Path exists via node 2: `|5 - 6| = 1 ≤ 2` and `|6 - 8| = 2 ≤ 2` → `true`.
- Query `[2,3]`: Direct edge exists because `|6 - 8| = 2 ≤ maxDiff` → `true`.

---

## Constraints

- `1 <= n == nums.length <= 10^5`
- `0 <= nums[i] <= 10^5`
- `nums` is sorted in **non-decreasing** order.
- `0 <= maxDiff <= 10^5`
- `1 <= queries.length <= 10^5`
- `queries[i] == [ui, vi]`
- `0 <= ui, vi < n`

---

## Solution Approach

### Key Insight

Because the graph is defined on a sorted array with edges based on absolute difference, **connectivity is determined only by adjacent elements**.

If `nums[i+1] - nums[i] > maxDiff`, then there is a **gap**. Any node to the left of the gap cannot connect to any node to the right, because all values are sorted and the difference between any left node and any right node is **at least** that gap, which is already greater than `maxDiff`.

Thus, we only need to connect adjacent indices `(i, i+1)` when their difference ≤ `maxDiff`. This is enough to determine all connected components.

### Algorithm

1. Initialize a **Union-Find (DSU)** data structure with `n` nodes.
2. Iterate over adjacent pairs `(i, i+1)`:
   - If `nums[i+1] - nums[i] <= maxDiff`, union nodes `i` and `i+1`.
3. For each query `(u, v)`, check if `find(u) == find(v)`.

### Complexity

- **Time:** `O(n + q * α(n))`, where `α` is the inverse Ackermann function (nearly constant).  
  - `O(n)` for building the DSU and scanning adjacent pairs.  
  - `O(q * α(n))` for answering all queries.  
- **Space:** `O(n)` for the DSU parent and size arrays.

---

## C++ Solution

```cpp
class Solution {
public:
    vector<bool> pathExistenceQueries(int n, vector<int>& nums, int maxDiff, vector<vector<int>>& queries) {
        vector<int> parent(n), size(n, 1);
        iota(parent.begin(), parent.end(), 0);

        function<int(int)> find = [&](int x) -> int {
            while (parent[x] != x) {
                parent[x] = parent[parent[x]];
                x = parent[x];
            }
            return x;
        };

        auto unite = [&](int a, int b) {
            int ra = find(a), rb = find(b);
            if (ra == rb) return;
            if (size[ra] < size[rb]) swap(ra, rb);
            parent[rb] = ra;
            size[ra] += size[rb];
        };

        // Only adjacent indices matter
        for (int i = 0; i + 1 < n; ++i) {
            if (nums[i + 1] - nums[i] <= maxDiff) {
                unite(i, i + 1);
            }
        }

        vector<bool> ans;
        ans.reserve(queries.size());

        for (const auto& q : queries) {
            ans.push_back(find(q[0]) == find(q[1]));
        }

        return ans;
    }
};
```


# 1288. Remove Covered Intervals

**Difficulty:** Medium  
**Topics:** Array, Sorting

---

## Problem Statement

Given an array `intervals` where `intervals[i] = [lᵢ, rᵢ]` represent the interval `[lᵢ, rᵢ)`, remove all intervals that are covered by another interval in the list.

The interval `[a, b)` is covered by the interval `[c, d)` if and only if `c <= a` and `b <= d`.

Return the number of remaining intervals.

---

## Examples

### Example 1

**Input:**  
`intervals = [[1,4],[3,6],[2,8]]`  
**Output:** `2`  
**Explanation:** Interval `[3,6]` is covered by `[2,8]`, therefore it is removed.

### Example 2

**Input:**  
`intervals = [[1,4],[2,3]]`  
**Output:** `1`  
**Explanation:** `[2,3]` is covered by `[1,4]`.

### Example 3 (Additional)

**Input:**  
`intervals = [[1,5],[2,6],[3,7],[4,8]]`  
**Output:** `4`  
**Explanation:** No interval covers another completely, so all remain.

---

## Approach

We can solve this problem efficiently with sorting and a single pass.

1. **Sort** the intervals:
   - Primary key: `start` ascending.
   - Secondary key: `end` descending.  
     This ensures that for intervals with the same start, the longer one appears first, making shorter ones easily identified as covered.

2. **Traverse** the sorted list while keeping track of the maximum `end` seen so far (`max_end`) among intervals that are **not** covered.
   - For each interval `[l, r]`:
     - If `r <= max_end`, this interval is completely covered by a previous interval → skip it.
     - Otherwise, it is not covered → increment the answer and update `max_end = max(max_end, r)`.

3. Return the count.

---

## Complexity

- **Time:** O(n log n) – due to sorting.
- **Space:** O(1) – aside from the input, only a few variables are used.

---

## Solution Code (Python)

```python
from typing import List

class Solution:
    def removeCoveredIntervals(self, intervals: List[List[int]]) -> int:
        # Sort by start ascending; if start equal, sort by end descending
        intervals.sort(key=lambda x: (x[0], -x[1]))
        
        count = 0
        max_end = -1
        
        for l, r in intervals:
            if r <= max_end:
                # This interval is covered by a previous one
                continue
            # Not covered, keep it
            count += 1
            max_end = max(max_end, r)
        
        return count
```

---

## Dry Run

For `intervals = [[1,4],[3,6],[2,8]]`:

- Sorted → `[[1,4], [2,8], [3,6]]`
- `[1,4]` → not covered, count=1, max_end=4
- `[2,8]` → 8 > 4, not covered, count=2, max_end=8
- `[3,6]` → 6 ≤ 8, covered, skip

Final answer = 2.

---

## Alternative Approaches

- **Brute force** O(n²) – compare every pair of intervals.
- **Sorting with stack** – similar to the above but using a stack to keep non‑covered intervals.

The sorting approach is optimal for this problem.

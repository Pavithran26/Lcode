
# 1331. Rank Transform of an Array

**Difficulty:** Easy  
**Topics:** Array, Hash Table, Sorting  
**Companies:** Amazon, Google, Microsoft

---

## Problem Statement

Given an array of integers `arr`, replace each element with its rank.

The rank represents how large the element is. The rank has the following rules:

1. Rank is an integer starting from `1`.
2. The larger the element, the larger the rank. If two elements are equal, their rank must be the same.
3. Rank should be as small as possible.

---

## Examples

### Example 1

**Input:**  
`arr = [40, 10, 20, 30]`  

**Output:**  
`[4, 1, 2, 3]`  

**Explanation:**  
40 is the largest element → rank 4  
10 is the smallest → rank 1  
20 is the second smallest → rank 2  
30 is the third smallest → rank 3

---

### Example 2

**Input:**  
`arr = [100, 100, 100]`  

**Output:**  
`[1, 1, 1]`  

**Explanation:**  
Same elements share the same rank.

---

### Example 3

**Input:**  
`arr = [37, 12, 28, 9, 100, 56, 80, 5, 12]`  

**Output:**  
`[5, 3, 4, 2, 8, 6, 7, 1, 3]`  

**Explanation:**  
Duplicate `12` gets the same rank (`3`).

---

## Constraints

- `0 <= arr.length <= 10^5`
- `-10^9 <= arr[i] <= 10^9`

---

## Approach

1. **Create a sorted copy** of the original array.
2. **Assign ranks** using a `HashMap`:
   - Iterate over the sorted array.
   - If a value is not already in the map, assign it the next available rank (starting at `1`) and increment the rank counter.
3. **Replace** each element in the original array with its corresponding rank from the map.
4. Return the transformed array.

**Time Complexity:** `O(n log n)` – due to sorting.  
**Space Complexity:** `O(n)` – for the sorted copy and the hash map.

---

## Java Solution

```java
import java.util.*;

class Solution {
    public int[] arrayRankTransform(int[] arr) {
        int n = arr.length;
        
        // Step 1: Sort a copy of the array
        int[] sorted = arr.clone();
        Arrays.sort(sorted);
        
        // Step 2: Assign ranks to unique values
        Map<Integer, Integer> rankMap = new HashMap<>();
        int rank = 1;
        for (int num : sorted) {
            if (!rankMap.containsKey(num)) {
                rankMap.put(num, rank);
                rank++;
            }
        }
        
        // Step 3: Replace original elements with their ranks
        int[] result = new int[n];
        for (int i = 0; i < n; i++) {
            result[i] = rankMap.get(arr[i]);
        }
        
        return result;
    }
}
```

---

## Edge Cases

- **Empty array** → returns `[]`
- **All elements equal** → all get rank `1`
- **Negative numbers** → handled correctly by natural ordering
- **Large constraints** → sorting in `O(n log n)` is acceptable for `10^5`

---

## Alternative (Using TreeSet)

If you want to avoid duplicates manually, you can use a `TreeSet` to get sorted unique values:

```java
TreeSet<Integer> set = new TreeSet<>();
for (int num : arr) set.add(num);

Map<Integer, Integer> map = new HashMap<>();
int r = 1;
for (int num : set) map.put(num, r++);

for (int i = 0; i < arr.length; i++) {
    arr[i] = map.get(arr[i]);
}
return arr;
```
